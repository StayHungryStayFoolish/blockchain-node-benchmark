# RPC Fixture Audit Summary

This directory keeps lightweight public audit summaries for fake-node RPC fixtures.

The full per-method request/response evidence generated by `tools/fake-node/record_rpc_fixtures.py`
can be large and is treated as a local audit artifact. It is ignored by Git under:

```text
docs/audit/rpc-fixtures/<chain>/<method>/
```

The committed fake-node runtime fixtures live under:

```text
tools/fake-node/fixtures/<chain>/*.json
```

`manifest.json` is generated by the authenticity validator and only lists
fixtures that have fresh per-method recording evidence. It is intentionally not
committed when the validator has no `real-recorded` entries; in that state it
would contain only an empty JSON array. Runtime fixture coverage is tracked by
`coverage-summary.json`, while recording evidence status is tracked by
`authenticity-report.json`.

Current committed coverage summary:

```bash
python3 tools/fake-node/check_fixture_coverage.py --json
```

For a local authenticity audit after recording fresh evidence, run:

```bash
tools/fake-node/record_rpc_fixtures.sh <chain>
python3 tools/fake-node/validate_fixture_authenticity.py --json
python3 tools/fake-node/check_fixture_coverage.py --json --strict
```
