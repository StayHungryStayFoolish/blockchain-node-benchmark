# Support

Use the right GitHub channel so maintainers can triage requests quickly.

## Bugs

Open a bug report issue when the framework behaves incorrectly and you can
provide reproduction steps, environment details, and relevant logs.

## Feature Requests

Open a feature request issue for new framework capabilities, report output,
monitoring behavior, Kubernetes support, or CI improvements.

## Chain or RPC Method Support

Open a chain/RPC support issue when you need a new chain, RPC method, parameter
shape, fixture, or fake-node behavior.

Include official documentation links and redacted request/response samples when
available.

## Questions and Usage Discussion

Use GitHub Discussions for installation questions, configuration questions,
roadmap discussion, and operational experience that is not yet a reproducible
bug or implementation-ready feature request.

## Security Issues

Do not open public issues for vulnerabilities, leaked secrets, private RPC
endpoints, or sensitive benchmark artifacts. Follow the process in
[SECURITY.md](SECURITY.md).
